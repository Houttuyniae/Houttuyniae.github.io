# 搜索镜像

docker search [OPTIONS] TERM

  • --automated : 只列出 automated build 类型的镜像；

  • --no-trunc : 显示完整的镜像描述；

  • -s : 列出收藏数不小于指定值的镜像。

docker search java

Docker 就会在 Docker Hub 中搜索含有 java 这个关键词的镜像仓库。

# 下载镜像

docker pull java:8

使用命令 docker pull 命令即可从 Docker Registry 上下载镜像，执行该命令后，Docker 会从 Docker Hub 中的 java 仓库下载最新版本的 Java 镜像。

如果要下载指定版本则在 java 后面加冒号指定版本

# 列出已下载的镜像

docker images

使用 docker images 命令即可列出已下载的镜像

在列出信息中，可以看到几个字段信息

  • REPOSITORY：表示镜像的仓库源

  • TAG：镜像的标签(版本)

  • IMAGE ID：镜像 ID

  • CREATED：镜像创建时间

  • SIZE：镜像大小

OPTIONS 参数

名称,简写 描述
-a 展示全部镜像(默认隐藏中间镜像)
-f 条件过滤
-q 只显示 id

# 删除镜像

docker rmi java

使用 docker rmi id 命令即可删除指定镜像

id 通过 docker images 中取得

docker rmi [ id | name ]

OPTIONS 参数

名称,简写 描述
-f 强制删除

删除全部镜像 docker rmi -f $(docker images -qa)

如果提示
Error response from daemon: conflict: unable to delete fce289e99eb9 (must be forced) - image is being used by stopped container 175b3f9741b7

先删除容器 docker rm <containerID>,在删除镜像

# 列出容器

docker container ls [options] 命令用于列出所有容器。

执行 docker container ls --help 查看相关属性

名称,简写 描述
--all, -a 显示所有容器(默认只显示运行的)
-f, --filter filter 根据提供的条件过滤输出
-n, --last int 显示最后创建的容器(包括所有状态)
-l, --latest 显示最新创建的容器(包括所有状态)
--no-trunc 不要截断输出
-q, --quiet 静默模式,只显示容器 ID
-s, --size 显示文件大小

# 启动容器

启动容器有两种方式，一种是基于镜像新建一个容器并启动，另外一个是将在终止状态（stopped）的容器重新启动。

因为 Docker 的容器实在太轻量级了，很多时候用户都是随时删除和新创建容器。

# 新建并启动

主要通过 docker run 命令

例如，从上述的 Hello World 实例 例子

利用 docker run 来启动容器，Docker 在后台运行的标准操作包括：

  • 检查本地是否存在指定的镜像，不存在就从公有仓库下载
  • 利用镜像创建并启动一个容器
  • 分配一个文件系统，并在只读的镜像层外面挂载一层可读写层
  • 从宿主主机配置的网桥接口中桥接一个虚拟接口到容器中去
  • 从地址池配置一个 ip 地址给容器
  • 执行用户指定的应用程序
  • 执行完毕后容器被终止

运行的部分参数

  • --name：容器名
  • -p：映射宿主主机端口 宿主机port:容器port
  • -v：挂载宿主目录到容器目录 [宿主机地址]:[容器内部地址]
  • -e：设置环境变量，此处指定 root 密码
  • -d：后台运行容器
  • --restart=always: 启动 docker 时也会启动
docker run
    --name mysql
    -p 3306:3306
    -v /home/mysql/data:/var/lib/mysql
    -v /home/mysql/conf.d:/etc/mysql/conf.d
    -e MYSQL_ROOT_PASSWORD=123456
    -d mysql:5.7.19

# 启动已终止容器

可以利用 docker start [长ID|短ID|name] 命令，直接将一个已经终止的容器启动运行。

# 删除容器

docker rm [container]

# 终止容器

docker stop [container]

终止所有容器

docker stop $(docker ps -a | awk '{ print $1}' | tail -n +2)

# 重启容器

docker restart [container]

-t：关闭容器的限时，如果超时未能关闭则用 kill 强制关闭，默认值 10s，这个时间用于容器的自己保存状态

# 进入容器

docker exec [OPTIONS] containerId COMMAND [ARG...]

docker exec -it 369bf1d952a6 /bin/bash
1

OPTIONS 参数

  • :分离模式: 在后台运行
  • -i :即使没有附加也保持 STDIN 打开
  • -t :分配一个伪终端
名称,简写 描述
-d 分离模式: 在后台运行
-i 即使没有附加也保持 STDIN 打开
-t 分配一个伪终端

# 退出容器

ctrl + p + q or exit

# 查询容器日志

docker logs [OPTIONS] CONTAINER

docker logs --since 4m --tail=10 -t aa0aae99f959cf6243338ce87c6f0f48cb6d1343f855d73ad53ef1999b453db0
1

OPTIONS 参数

名称,简写 描述
-f 跟踪日志输出
--since 显示某个开始时间的所有日志
-t 显示时间戳
--tail 仅列出最新 N 条容器日志
